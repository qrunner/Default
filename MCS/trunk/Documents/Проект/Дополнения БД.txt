Сгруппировал таблицы по представляемым объектам.
Существующие таблицы пометил знаком "*". Новые записи обозначил.
Запись (-> TABLE_NAME) напротив поля означает, что в этом поле стоит
идентификатор таблицы с именем TABLE_NAME.


I. Информационное представление меню.

*  1. Категории:   MCATEGORY

      1. id            идентификатор
      2. parent_id     идентификатор содержащей категории  (-> MCATEGORY)
      3. name          название
      4. place_id      идентификатор заведения  (-> PLACE)

*  2. Блюда:   DISH

      1. id               идентификатор
      2. mcategory_id     категория      (-> MCATEGORY)
      3. name             название
      4. description      описание
      5. price            цена
      6. pict             изображение                 новое поле

*  3. Свойства блюд:   TAG

      1. id           идентификатор
      2. name         название
      3. pict         изображение  (иконка)         новое поле    
      4. place_id     (-> PLACE)                    новое поле

   4. Ингредиенты:   INGREDIENT
 
      1. id           идентификатор        
      2. name         название
      3. place_id     (-> PLACE)

   5. Состав блюда:   DISH_COMPOSITION

      1. dish_id         (-> DISH)
      2. ingredient_id   (-> INGREDIENT)

*  6. Состав свойств блюда: DISH_TAGS

      1. dish_id         (-> DISH)
      2. tag_id          (-> TAG)

II. Информационное представление новостей и акций.

    1. Новости:   NEWS
    
       1. id              идентификатор
       2. news_date       дата
       3. title           заголовок
       4. short_text      краткое содержание
       5. full_text       полное содержание
       6. pict            изображение
       7. place_id        заведение           (-> PLACE)

    2. Акции:   ACTION

       1. id             идентификатор
       2. start_date     начальная дата
       3. end_date       конечная дата
       4. name           название
       5. text           описание
       6. pict           изображение
       7. place_id       заведение       (-> PLACE)

III. Служебные.

    1. Функции пользователя:   USER_FUNCTION

       1. id          идентификатор
       2. name        название
       
    2. Пользователи:   USER

       1. id          идентификатор
       2. l_name      фамилия
       3. f_name      имя
       4. m_name      отчество
       5. place_id    заведение   (-> PLACE)
       
    3. Назначение функций пользователям:   USER_ROLE
